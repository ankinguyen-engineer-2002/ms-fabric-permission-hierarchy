── Data-layer security (song song, khong thay the nhau) ──

Permissions ho tro

Read (table/folder) ReadWrite (Preview) RLS (row filter) CLS (column list) OLS (table/schema)

Combination logic

∪Multiple roles → UNION (least-restrictive). Role A full table + Role B RLS = full access wins
∩Trong 1 role: RLS ∩ CLS ∩ OLS (intersection)
ORNhieu roles cung RLS: predicate noi OR

Luu y quan trong

❌Admin/Member/Contributor bypass hoan toan — chi Viewer bi enforce
⚠Khi enable: DefaultReader role auto-tao. Phai xoa hoac remove ReadAll khoi user cu
⚠Khong the tat sau khi bat — per-item opt-in
✓Ap dung dong nhat: Spark, SQL endpoint, Power BI, OneLake API

Cong cu T-SQL

GRANT / REVOKE DENY (override GRANT) CREATE SECURITY POLICY (RLS) GRANT SELECT (column list) ALTER TABLE MASKED (DDM) Views · SPs · Functions

Khac biet vs OneLake Security

⚡DENY override GRANT — nguoc voi OneLake (most permissive wins)
✓DDM ho tro — OneLake Security khong co DDM
⚠Workspace Admin bypass T-SQL DENY
✓Views/SPs co the dung SQL GRANT doc lap voi table permission

Security types

RLS (DAX row filter) OLS (hide table/column) CLS (column restrict)

Direct Lake mode gotcha

⚡Direct Lake: RLS enforce tai semantic model — khong di qua OneLake Security
⚠DirectLake over SQL mode: identity owner's credentials — khong pass user identity qua shortcut
✓DirectLake over OneLake mode: user identity pass through dung

Build permission risk

❌User co Build permission → connect truc tiep vao dataset → bypass RLS cua report
✓RLS enforce o model level van ap dung ke ca khi build custom report
⚠OLS an table/column khoi schema — khong han la security layer that su
